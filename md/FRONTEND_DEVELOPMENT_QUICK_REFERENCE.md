## 💻 Git Workflow

```bash
# Start a feature
git checkout -b feature/search-filtering
git add .
git commit -m "feat: add search and price filtering"

# Daily commits
git add src/components/product/FilterPanel.tsx
git commit -m "feat: add price range slider"

# Push to origin
git push origin feature/search-filtering

# Create PR for review
# → Team reviews code
# → Merge to main
# → Start new feature

git checkout main
git pull origin main
git checkout -b feature/wishlist
```

## 🎯 Definition of Done (Each Component)

```
A component is "done" when:

Code Quality ✓
├─ TypeScript types defined
├─ No console errors
├─ No ESLint warnings
└─ Code formatted (Prettier)

Functionality ✓
├─ Feature works as spec
├─ All edge cases handled
├─ Error states handled
└─ Loading states shown

Testing ✓
├─ Manual testing passed
├─ Mobile testing passed
├─ Browser testing passed
└─ API integration verified

Documentation ✓
├─ Comments on complex code
├─ Props documented
├─ Usage examples provided
└─ Known issues documented

Performance ✓
├─ No console warnings
├─ Images optimized
├─ Lazy loading if needed
└─ Component re-renders optimal
```

## ⚡ Quick Start Commands

```bash
# Setup
cd /workspaces/localcart/frontend
npm install

# Development
npm run dev                    # Start dev server

# Testing
npm run lint                   # Check code quality
npm run type-check             # TypeScript check

# Building
npm run build                  # Build for production
npm start                      # Run production build

# Deployment
vercel --prod                  # Deploy to production

# Common tasks
git status                     # See what changed
git log --oneline              # See recent commits
git diff                       # See exact changes
```
